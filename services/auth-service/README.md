# Auth Service

The Authentication & Authorization microservice for the Production-Grade E-Commerce Platform.

## Features
- User Registration and Login
- Stateless JWT generation and validation
- Refresh Token management via Redis
- Role-Based Access Control (RBAC)
- Password encryption using BCrypt
- User Profile management

## Architecture
This service uses a PostgreSQL database for persistent user records and a Redis cluster for transient refresh tokens. It acts as the central authority for identity. All other microservices rely on the API Gateway to validate the JWTs generated by this service.

## Configuration
Requires the following environment variables:
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`
- `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`
- `JWT_SECRET` (Must be highly secure and at least 256 bits).

## Running Locally
```bash
mvn clean install
mvn spring-boot:run
```
To run tests:
```bash
mvn test
```
*(Integration tests use Testcontainers and require a running Docker daemon).*
